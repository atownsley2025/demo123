ZEROSKID

Wet-Road Detection with Deep Learning

ZeroSkid is a concept for an assistive driving system that uses computer vision and AlexNet to detect wet pavement in real time. The idea started from seeing how often rain and standing water lead to crashes that feel avoidable if drivers had just a little more warning.

See the Problem & Solution How the Tech Works

Problem & Solution Overview

The Problem

Wet pavement is a quiet but serious hazard. It contributes to a large number of crashes each year and brings major physical, emotional, and financial costs.

  • ABS and traction control only react after a skid starts.
  • Drivers can’t always see thin water layers or puddles, especially at night.
  • Most cars don’t have a system focused specifically on wet-road detection.

The ZeroSkid Idea

ZeroSkid imagines a small dash-mounted device with a camera and onboard model that constantly looks at the road ahead and flags when it appears wet or slick.

  • Real-time classification of the lane as dry or wet.
  • Short alert sound or icon when wet pavement is detected.
  • Focus on being simple, affordable, and add-on friendly for existing cars.

Tools & Technical Approach

Core Tools

  • AlexNet – deep convolutional neural network for image classification.
  • Python & PyTorch (or similar) for training and experimenting with the model.
  • Google Colab for running training code and visualizing results.
  • GitHub Pages for hosting this project website.

Why AlexNet?

AlexNet is a good fit because its convolutional layers learn to recognize textures, reflections, and edge patterns that change when pavement is wet.

Architecture Summary

  • 5 convolutional layers for feature extraction.
  • 3 fully connected layers for final classification.
  • ReLU activations and max pooling to reduce noise and downsample.
  • Softmax output layer that predicts wet vs. dry.

Earlier layers focus on simple shapes and brightness changes; deeper layers respond to more complex textures and reflections on the road surface.

Data, Training & Results

Training Setup

Input data: Labeled wet & dry road images
Loss: Cross-entropy loss
Optimizer: SGD or Adam
Learning rate: ~0.001
Batch size: 32
Epochs: 10–30 (depending on dataset size)

Preprocessing

  • Resize to 224×224×3 (AlexNet input size).
  • Normalize pixel values for stable training.
  • Apply brightness tweaks, flips, and rotations as augmentation.
